# Spring AI 编码侧的「可理解性」：Tepeu 上一次可复现的初步验证

> **本文是什么：** 单一仓库（Tepeu / Spring AI）上的工程札记 + ATE 对照；主张可被数据打脸，且已被部分打脸。  
> **本文不是什么：** 跨产品定律、客服准确率证明、或对 PithTrain 的复现论文。  
> 原始数据：`experiments/ate-bench/results/MERGED-20260715.md`（含历史单场与全部目录）。  
> **主读者：** 维护 Spring AI / 工具面常改的工程师——要选型，不是口号。  
> **快读：** 信 **§5.0**。H1′：通道随模型。glm 上 C（只注册）×3 合计≈**-12%**，A+B≈**-25%**——注册有红利但**解释不了全部**；多出的主要在 T2。flash 上 C 伤 T3。注入：DeepSeek 有效、glm 无效。A+B：flash≈**-4%** / glm≈**-25%**。理解类注入 n=10≈11.6。出口：**§6**。

## 1. 引言

维护者变成「人 + 编码 Agent」后，Spring 的间接层（DI、注解扫描、框架回调）会变成 Agent 的检索税：

> 加工具要翻多少轮？路径说明放进仓库会不会根本没人读？

本文用本仓数据钉一句：

> **对 Agent 友好 = 写进结构 + 写进模型会用的通道；多一份等人来翻的 Markdown，不算信息策略。**

灵感来自 PithTrain 一类实践；**方法与规模不对齐**。贡献是 Tepeu 消融与边界，不是「移植成功」。

**读法：** §2–4 概念（可快翻）→ **§5.0** → **§6**。§5 其余与 `MERGED` 是附录。§4.2 幂等/记忆 **无 ATE**。

## 2. 两个「Agent」，勿偷换

| | 编码辅助 Agent | 业务执行 Agent |
|--|----------------|----------------|
| 任务 | 改代码、定位 bug | 跑业务流程 |
| 本文回答 | 系统好不好改（turns） | **不回答** KPI / 准确率 |

编码侧显式注册，最多给值班「允许集里有哪些工具」的**间接**可解释性；**绝不能**用 turns 降幅宣称客服准确率上升。业务硬约束见 §4.2。

---

## 3. 对人类友好 ≠ 对 Agent 友好

人类靠模式识别消化 Spring；Agent 靠逐字追踪拼路径。同一套默认「糖衣」，常变成后者的检索税。

Tepeu 里一条工具聊天：人脑是 Controller → Orchestrator → ChatService → 工具；Agent 要串 `ChatController` / `AgentOrchestrator` / `ChatService`+`ToolCallbacks` / 各 `*Tools` / 框架内 `ToolCallingAdvisor`。静态阅读无法一次闭合。

**基础设施可保留抽象；工具边界与决策核应优先显式。**  
地图：人要示意图，Agent 要施工图——且施工图得交到**它会用的手里**（H1′：system 或仓库检索，随模型），不能只锁在资料室。

## 5. 数据

### 5.0 最终主张（只信这张）

同一 Claude Code CLI；改仓库形态 / system 内容；看 `num_turns`。数字是**量级感**，不是置信区间。

| # | 主张 | 信什么 | 勿信成 |
|---|------|--------|--------|
| 1 | 注册 alone（C）不够当全部故事 | flash：同日 T3 变差；**glm ×3：合计≈-12%，远小于 A+B≈-25%** | 「有注册表就够」或「glm 红利全是注册」 |
| 2 | 被动 md ≠ 信息策略 | T1：有文档 turns 不降（flash 主锚） | 「写了 AGENT_*.md = 投喂」 |
| 3 | 主动注入降**理解税** | T1 inject **n=10**，均值 ≈11.6（被动 ≈21） | 任意模型任意仓 |
| 4 | 注入降**生成税** | flash/pro：A-only/样例降 T3；**glm：未降** | 跨厂银弹 |
| 5 | A+B 少 turns | **仅认 ×3：** flash≈**-4%**；glm≈**-25%** | 历史 -35% / 单场 -24% |
| 6 | 调试/重构别默认 A+B | T6/T8 flash 更差；T7≈持平；pro/glm 的 T6 相反（单轮） | 「显式化全面加速」 |

**最终信哪张表：** A+B → #5；glm 上 C vs A+B → #1 + 下节；注入 → #3/#4。

#### H1′：通道偏好（已用 glm×C×3 收窄）

| 模型 | system 注入 | C 只注册 | A+B（注册+仓库 md） |
|------|-------------|----------|---------------------|
| flash | T3 降 | 同日合计≈-9%，**T3 变差** | ×3 合计≈**-4%** |
| glm-5.2 | T3 **不降** | ×3 合计≈**-12.4%**（T3≈A+B，**T2 几乎无增益**） | ×3 合计≈**-24.6%**（T2+T3 同向） |

> **机制句：** 降税的不是载体口号，而是图景是否进入**该模型的解题轨迹**。  
> **glm×C×3 裁决（中间结果，两边都修）：** C 有效但**达不到** A+B（-12% vs -25%）。T3 上 C≈A+B（均≈17.3 turns）→ 生成型红利**大部分来自注册**，且与 flash「C 伤 T3」相反。T2 上 C≈Baseline、A+B 明显更好 → **多出的合计降幅主要来自路径 md 对检索/列文件的帮助**。故既不能写成「glm 红利全是 md」，也不能写成「全是注册」。  
> **产品：** 注册仍必做（glm 上对 T3 也值）；要不要仓库 md，看本模型 T2/扩展检索是否吃文档——冒烟再接线。  
> **可检验命题（机制声明）：** DeepSeek 更依赖 system 注入的训练先验；glm 对 system dump 钝、对仓库侧结构更敏——其中**注册解释 T3，路径 md 额外解释 T2**。下一步可检验：glm 上「有 md、无新注册结构」或「注入改写成检索友好形态」是否移动 T2/T3。这不是终点，是可被修正的起点。

#### 外推边界（统计诚实）

- 1 代码库；主锚 flash；pro/glm 为对照。  
- A+B / glm×C / T6–T8 多为 **×3**；flash 上 C、A-only 等仍有单场。  
- glm A+B 批次曾 **API 529**（已重试）；glm×C 批次无 529。  
- 外推：换仓、换厂、要效应量 CI → 须另跑；本文只支撑**本仓选型 + 冒烟习惯**。

## 6. 决策树（本仓证据，非定律）

```
改动频率？
├─ 每周多次改工具/调用链
│   └─ 显式注册必做；再对本模型冒烟：system 注入 vs 仓库 md（H1′），赢家进 CI
├─ 每月数次
│   └─ 显式注册；注入/md 按多人共用与冒烟结果取舍
└─ 每季或更少
    └─ 结构可不动；改前临时把路径贴进 prompt / 工作区

任务类型？（已注册）
├─ 扩展·检索型（近 T2）→ 对本模型冒烟：注册 alone vs +路径 md（flash 上注册常够；glm 上 md 额外拉 T2）
├─ 扩展·生成型（近 T3）→ 分别冒烟：注入 vs 被动 md（勿默认 A-only；glm 上 T3 多吃注册）
├─ 理解调用链（近 T1）→ 优先测注入；勿假设「有 md 就会读」
├─ 调试（近 T6）→ 勿默认 A+B 降 turns
├─ 重构（近 T7）→ 本仓 ≈持平
└─ 琐碎常量（近 T4）→ 不必改架构

同步债？
├─ 启用了 inject → generate_call_path_inject.py + check --mode fail
└─ 否则 → check --mode degrade，过期则停用 inject
```

**可做：** 按树选型；本仓前后测；§4.2 另做，别指望 turns 证明。  
**不可做：** 引 -35%/-24% 当稳定效应；「永远注入」或「永远放 md」；注册 alone=全部故事；H1′→跨厂定律；编码 turns→客服准确率。

---

> **收束：** 核心是 **H1′**：通道随模型；glm 上注册解释大部分 T3、路径 md 额外拉 T2，故 A+B＞C＞≈0。理解类注入 n=10 仍最稳。注入/A+B/C 都带模型条件。调试勿默认红利。继续打脸时改假设，不改诚实。
