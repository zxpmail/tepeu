# Workflow — Domain Mapper

> 完整工作流，5 步。每一步完成后暂停，让用户决定继续还是调整方向。

---

## 前置：选择深度级别

进入 Step 1 前，确认用户想要的深度：

```
L1 Snapshot → 只走 Step 2，输出 domain-map.md（~30 分钟）
L2 Standard → Step 2 → Step 3，输出 domain-map.md + competitor-analysis.md（~1-2h）
L3 Deep    → Step 2 → Step 3 → Step 4 → Step 5，全量输出（~半天）
```

**如果用户不确定**：默认 L2。跑完 Step 2 后可以升级。

---

## Step 1: 定界（Scope Definition）

**目标**：把模糊的"帮我研究 X"变成可执行的边界条件。

### 执行

1. **域过宽检测**：如果用户给的领域明显太宽（"科技行业"、"AI"、"半导体"），强制收窄：
   - "半导体太大了——芯片设计 / 制造 / 封装 / 设备，你关心哪一层？"
   - "AI 也太宽了——模型训练 / 推理芯片 / 应用层 / 数据基础设施？"

2. **边界确认**：明确以下内容：
   - 领域名称（domain-map.md 文件名）
   - 关注的上下游范围
   - 深度级别（L1 / L2 / L3）
   - 是否有已知信息源（如有，优先利用；如无，声明信息来源不可控）

3. **现有地图检查**：如果 domain-map.md 已存在：
   - 读取文件头部的生成时间戳
   - 超过 30 天 → 提示老化，询问是否重新生成
   - 30 天内 → 询问是追加还是覆盖

4. **输出**：确认好 scope 后进入 Step 2

### 失败模式

| 条件 | 处理 |
|------|------|
| 用户坚持宽域（"整个半导体都要"） | 接受，但在输出头部标注 ⚠️ "覆盖范围极宽，深度不足" |
| 用户说"随便" | 建议一个热门子领域，确认后继续 |
| 领域过于小众，WebSearch 0 结果 | 告知信息来源不足，建议用户提供种子链接 |

---

## Step 2: 领域快照（Domain Snapshot）

**目标**：输出 `domain-map.md`，覆盖 10 个标准维度。

### 执行

1. **并行搜索**（按维度分组，减少来回）：
   - 玩家格局：主要品牌/公司，定位一句话，核心优势
   - 产品/技术分代：代际演进，关键特征，代表产品
   - 下游客户结构：谁在买，买来做什么，决策链
   - 商业模式：怎么赚钱的，定价权在哪，成本结构
   - 痛点和矛盾：用户/买家抱怨什么，行业瓶颈
   - 社媒生态：Reddit 板块 / YouTube 频道 / Twitter 账号
   - 关键词和术语：行业黑话，搜索高频词

2. **Surprise Me 标注**：在输出中显式标注 "⭐ 意外发现"——跟在其他行业的同类公司身上看不到的信息。这些是后续深挖的候选入口。

3. **源标注**：每条关键输出附来源 URL，格式：
   ```
   - SK hynix 在 HBM3E 量产上领先 [来源](https://...)
   - ⚠️ 三星 HBM4 量产时间尚未官方确认 [来源不足]
   ```

4. **输出**：按 `templates/domain-map-template.md` 写入 `domain-map.md`

5. **暂停**：展示给用户，问：
   - "有没有哪个方向你想深挖？"
   - （如果有 ⭐ 标注）"这几个意外发现里，哪个你最感兴趣？"
   - "继续到 Step 3 还是够了？"

### 失败模式

| 条件 | 处理 |
|------|------|
| 某个维度完全搜不到信息 | 保留维度标题，内容标注 ⚠️ "公开信息不足" |
| 不同来源信息矛盾 | 并列展示矛盾双方，标注各自来源，不强行统一 |
| 搜到的信息明显过时（>1 年） | 在条目旁标注 📅 日期 |

---

## Step 3: 深度潜水（Deep-Dive）

> 仅 L2 / L3。对 Step 2 中确定的重点实体逐个分析。

**目标**：输出 `competitor-analysis.md`（对比表）。

### 执行

1. **确认深潜对象**：从 Step 2 输出中选定 2-5 个关键实体（通常是主要玩家）

2. **逐实体分析**（对每个实体）：
   - **官网结构**：导航菜单、产品分类、技术规格页结构
   - **对外话术**：品牌 slogan、首页价值主张、About Us 叙事
   - **产品线**：分类方式、定价层级（如有）、技术规格对比
   - **IR / 投资者页面**：财报重点、增长叙事、风险提示
   - **技术内容**：白皮书、技术博客、文档站点结构

3. **对比表合成**：所有实体分析完成后，输出对比表：
   ```
   | 维度 | SK hynix | Samsung | Micron |
   |------|----------|---------|--------|
   | 对外叙事 | AI Memory | 技术代际 | 供应链安全 |
   | 主打产品 | HBM3E | HBM4 定制 | 美国本土 |
   | 客户策略 | 绑定 NVIDIA | 绑定 ASIC | 分散 |
   | 定价策略 | 良率溢价 | 追赶定价 | 地缘溢价 |
   ```

4. ⭐ **叙事差异分析**：三家公司官网讲的是不是同一个故事？如果不是——这暴露了行业的分化点。

5. **输出**：写入 `competitor-analysis.md`

6. **暂停**：问用户 "继续到 Step 4（社媒）还是够了？"

### 失败模式

| 条件 | 处理 |
|------|------|
| 官网无法访问 | 标注 ⚠️，改用 WebSearch + 第三方报道 |
| 某实体信息远少于其他实体 | 标注信息不对称，暗示竞争情报信号（要么透明要么隐藏） |
| 官网话术高度雷同 | 标注"差异化不足"——这本身就是重要发现 |

---

## Step 4: 社媒监听（Social/Media Analysis）

> 仅 L3。分析社媒内容、识别关键意见领袖和热点话题。

**目标**：输出 `social-analysis.md`

### 执行

1. **渠道映射**：从 Step 2 的社媒生态列表出发：
   - YouTube：关键频道、内容类型（技术解析 / 行业预测 / 新闻追踪 / 观点输出）
   - Twitter/X：关键账号、观点阵营、互动模式
   - Reddit：主要板块、热帖类型、社区情绪

2. **内容分析**（对每个渠道）：
   - 点赞/转发/评论最高的内容
   - 分类：技术解析型 / 行业预测型 / 新闻追踪型 / 观点输出型
   - ⭐ **什么内容能爆，为什么爆，谁在爆**

3. **核心矛盾提取**：
   - 社区里讨论最热的两个问题是什么
   - 这两个问题的答案 = 行业核心矛盾
   - 多方观点并列（不要只取一方）

4. **输出**：写入 `social-analysis.md`

5. **暂停**：问用户 "要不要合成最终路线图？"

### 失败模式

| 条件 | 处理 |
|------|------|
| 平台限制无法爬取 | 标注无法访问的平台，基于 WebSearch 二手信息 |
| 社媒热度极低 | "这个领域未进入大众讨论"——本身就是信号（太早期 / 太 niche） |
| 信息噪音大 | 过滤纯情绪帖，只保留 3 类内容：爆料、分析、预测 |

---

## Step 5: 合成（Synthesis）

> 仅 L3。把前三步的输出压成一张路线图。

**目标**：更新 `domain-map.md`，追加核心矛盾、路线图和风险表。

### 执行

1. **核心矛盾提炼**：
   - 从 Step 2-4 中提取行业里争论最热的 2-3 个根本问题
   - 每个矛盾用"一方面……另一方面……"格式陈述

2. **路线图（树状结构）**：
   ```
   <领域>
   ├─ 产品代际
   │  ├─ 当前主力
   │  └─ 下一代（量产中）
   ├─ 供应商
   │  ├─ 领先者
   │  ├─ 追赶者
   │  └─ 跟跑者
   ├─ 下游
   │  ├─ 主要客户
   │  └─ 新兴市场
   └─ 风险
      ├─ 技术瓶颈
      ├─ 产能瓶颈
      └─ 定价权博弈
   ```

3. **风险表**：

   | 风险 | 影响对象 | 概率 | 早期信号 |
   |------|---------|------|---------|
   | ... | ... | ... | ... |

4. **已知盲区**：明确列出"以下问题当前公开信息无法回答"——比假装全知道更有价值。

5. **输出**：追加到 `domain-map.md`

6. **完成**：问用户 "要保存还是发给谁？"

---

## 全流程摘要

```
Step 1: 定界
  └─ 域过宽收窄 / 深度级别 / 现有地图老化检查
  ↓
Step 2: 领域快照 → domain-map.md
  └─ 10 维度并行搜索 / Surprise Me 标注 / 源标注
  └─ ⏸ "深挖哪个方向？"
  ↓  (L2/L3)
Step 3: 深度潜水 → competitor-analysis.md
  └─ 逐实体官网分析 / 对比表 / 叙事差异分析
  └─ ⏸ "继续社媒？"
  ↓  (L3)
Step 4: 社媒监听 → social-analysis.md
  └─ 渠道映射 / 内容分析 / 核心矛盾提取
  └─ ⏸ "合成路线图？"
  ↓
Step 5: 合成 → domain-map.md 更新
  └─ 核心矛盾 / 路线图 / 风险表 / 已知盲区
  └─ ✅ 完成
```
