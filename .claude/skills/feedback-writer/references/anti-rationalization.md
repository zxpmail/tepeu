# Anti-Rationalization Checklist（feedback-writer）

| Rationalization | Reality |
|---|---|
| "This is just a minor correction, no need to record feedback" | Every correction is a learning opportunity. If not recorded, the same failure repeats. Small entries still count toward evolution thresholds. |
| "I'll record it later when I have more context" | You won't. Record it now or lose the signal. Context can be updated later. |
| "Skip scoring, the qualitative description is enough" | Score-less feedback can't trigger evolution thresholds. Always fill Precision/Coverage/Efficiency/Satisfaction. |
| "This frustration is clearly feedback worth recording" | Frustration at the tool/environment/language is not AI capability feedback. Discriminate signal from noise. |
| "This is different enough for a new entry" | Check FEEDBACK-INDEX.md first. If a similar topic exists, merge — don't duplicate. Same failure recorded 5 times inflates counts. |
| "I don't know the failure_class, so I'll leave it unset" | `unset` is valid, but add a sentence explaining ambiguity. Better than guessing, but worse than determining it. |
