# Evolution Proposal Dimension Checklist

[Dimension Checklist]

| Tier | Dimension | Criteria |
|------|-----------|----------|
| **Must-Have** | Data Support | ≥3 feedback entries supporting the pattern; single entry is an anecdote, not a signal |
| **Must-Have** | RED Observation | Specific Agent behavior cited from feedback excerpt or session log; not intuition |
| **Must-Have** | GREEN Change | Exact target file + section + text to add or change; "update the Skill" is insufficient |
| **Must-Have** | Verify-By | Observable, time-bounded criterion to confirm fix; falsifiable |
| **Must-Have** | Failure Classification | failure_class identified (skill-defect / execution-lapse / unset); determines routing |
| **Recommended** | Cross-Reference | Pattern cross-referenced against task-history.md to rule out false correlation |
| **Recommended** | Denominator Check | Skill usage count checked before acting on scores; 5 uses at 60% is noise |
| **Recommended** | WebSearch | Searched for existing best practices before proposing new rules or Skills |
| **Recommended** | Effort Estimate | Size estimate (small/medium/large) attached to each proposal |
| **Optional** | Before/After Diff | Example diff in proposal showing what changes |
| **Optional** | Score Trend | Skill score trend (Precision/Coverage) attached when applicable |
