# Anti-Rationalization Checklist（skill-builder）

[Anti-Rationalization Checklist]

| Rationalization | Reality |
|---|---|
| "I know the structure well enough, no need to read the template" | Every time you skip the template, you'll miss something. Read it anyway — takes 30 seconds. |
| "This Skill is unique, no need to reference existing ones" | Uniqueness of domain doesn't justify unique format. Match existing Skills for consistency. |
| "I'll fill empty sections later" | You won't. Empty sections cause confusion when the Skill is invoked. Either fill it now or omit it. |
| "The domain is simple, no need for WebSearch" | Simple domains still benefit from established best practices. Search first, write second. |
| "Trigger description is clear enough" | Clear to you is not clear to the dispatcher. Test with triggers.json near-miss cases before shipping. |
